postman logic 
Short step-by-step (exact, minimal):

1. **Start backend**

   * `uvicorn main:app --reload` (or however you run it). Server on `http://localhost:8000`.

2. **Check current model**

   * GET `http://localhost:8000/feedback/get_model`
   * Expect: `{"current_model":"vader"}`

3. **(Optional) Change model**

   * POST `http://localhost:8000/feedback/set_model`
   * Body ‚Üí JSON: `{ "model": "huggingface" }` (allowed: `"vader"`, `"huggingface"`, `"tl_model"`)
   * Expect: `{"message":"Model switched to huggingface"}`

4. **Prepare input**

   * Option A ‚Äî CSV: first column = review text, optional second column = `YYYY-MM-DD` timestamp. (Header row OK ‚Äî it will be skipped.)
   * Option B ‚Äî TXT: plain lines, one review per line.
   * Option C ‚Äî JSON array: raw body `["review1", "review2", ...]`

5. **Run analysis**

   * POST `http://localhost:8000/feedback/analyze`
   * If using file: choose `form-data` ‚Üí key `file` ‚Üí type `file` ‚Üí attach `.csv` or `.txt`.
   * If using JSON array: set `Body` ‚Üí `raw` ‚Üí `JSON` and paste array.
   * **Important:** `analyze` uses the backend‚Äôs current model (set by `/set_model`). Do **not** expect a `?model=...` query param to be honored.

6. **Expected JSON response (keys)**

   * `sentiment_distribution` (pos/neg/neu counts + percentages)
   * `per_review_summary` (list of `{id, text, sentiment, compound}`)
   * `aspect_sentiment` (counts per aspect)
   * `top_issues` (keywords with count & percent)
   * `suggestions` (list of suggestions)
   * `impact_score` (0‚Äì100)
   * `confidence_overview` (average, median, histogram)
   * `wordcloud_data` (positive & negative word frequencies)
   * `trend_over_time` (date-wise pos/neg/neu)
   * `report_download` (a **base64 CSV string** you can decode)

   Example (short): `{"sentiment_distribution":{...},"per_review_summary":[...],"report_download":"UmV2aWV3..."}`

7. **Download CSV report**

   * POST `http://localhost:8000/feedback/report/download`
   * Body JSON: `{ "report_base64": "<value from response.report_download>" }`
   * Response: binary file stream ‚Äî in Postman click **Save Response** to store as `.csv`.

8. **Errors you'll see**

   * `400` if no input provided or unsupported file type.
   * `400` if `/set_model` receives invalid model name.
